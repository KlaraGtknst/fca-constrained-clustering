\subsection{Topic Model}
\label{sec:topic_model}
The data-driven topic model-based approach comprises three steps: (i) constructing a \ac{lda} topic model, (ii) deriving a \textcolor{orange}{constrained} document-topic incidence relation, and (iii) filtering concepts using the TITANIC algorithm~\cite{gutekunst2025conceptual}.

\subsubsection{Topic Model Construction}
We employ \texttt{gensim}'s implementation of \ac{lda}\footnote{\url{https://radimrehurek.com/gensim/models/ldamodel.html} (30.12.2025)}~\cite{alghamdi_survey_2015}, a widely used generative probabilistic model that represents documents as mixtures of $K$ latent topics.
\textcolor{orange}{To ensure comparability with the manually annotated BankSearch taxonomy, the number of latent topics is fixed to the number of ground-truth categories ($K=11$).}
For a corpus of $N$ documents, \ac{lda} produces a document-topic probability matrix $\Theta \in \mathbb{R}^{N \times K}$, where $\theta_{dt}$ denotes the probability that document $d$ is associated with topic $t$.
To derive a binary formal context from $\Theta$, document-topic probabilities are thresholded using a minimum topic probability $\delta$.
The threshold is selected empirically using the elbow criterion on the resulting incidence density, as illustrated in Figure~\ref{fig:incidence_density} for the BankSearch dataset.

\subsubsection{Document-Topic Incidence}
Given a threshold $\delta$, a document $d$ is assigned topic $t$ if $\theta_{dt} \geq \delta$. 
This produces a binary incidence relation $I \subseteq D \times T$, where $(d, t) \in I$ indicates that topic $t$ sufficiently characterizes document $d$. 
Lower threshold values yield denser contexts with more document-topic associations, whereas higher values retain only the most dominant topics.
If no topic exceeds the threshold, the highest-probability topic is assigned to ensure that each document remains associated with at least one topic.

The resulting binary incidence relation $I \subseteq D \times T$ defines the formal context $\mathbb{K}=(G,M,I)$, where $G$ denotes the set of documents (objects), $M$ the set of topics (attributes), and $I$ the document-topic incidence relation.

\subsubsection{Concept Filtering via TITANIC}

Computing the complete concept lattice of $\mathbb{K}$ can be computationally challenging, as the number of concepts may grow exponentially with the size of the context. 
Moreover, large lattices often hinder interpretability.
To address this issue, Stumme et al.~\cite{stumme_titanic_2002} introduced the TITANIC algorithm, which computes \emph{iceberg concept lattices}. 
These lattices contain only concepts whose intents correspond to frequent attribute sets.

Let $\mathtt{min\_supp} \in [0,1]$ denote a minimum support threshold. 
An attribute set $B \subseteq M$ is considered frequent if $\operatorname{supp}(B) = \frac{\left| B' \right|}{\left| G \right|}$ meets or exceeds $\mathtt{min\_supp}$. 
A concept $(A,B)$ is retained in the iceberg concept lattice if its intent $B$ is frequent.

Intuitively, this restriction preserves only those topic combinations that characterize at least $\mathtt{min\_supp} \times 100\%$ documents of the corpus. 
The choice of $\mathtt{min\_supp}$ directly influences the granularity of the resulting hierarchy.
Figure~\ref{fig:concepts_support} shows the relationship between the minimum support threshold and the number of extracted concepts for the BankSearch dataset. 
% Based on this analysis, we select $\mathtt{min\_supp} = 0.05$, yielding 30 concepts while maintaining interpretability.
